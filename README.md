# astrbot_plugin_qweather

基于和风天气 API 的 AstrBot 个人天气与通勤建议插件。

## 命令

- `/天气 [城市]`：按天气日期切换时间自动查询今日或明日详细天气、天气简报、穿衣和生活建议。
- `/通勤`：根据当前时间自动判断输出今日/明日上班、下班天气和交通建议。
- `/天气帮助`：查看可用命令和配置提示。
- `/预警`：手动查询当前生效天气预警，无预警时也会回复。
- `/预警检查`：定时检查入口，仅在有新增或更新预警时输出。

## 配置

所有参数均通过 AstrBot 插件配置页面填写，插件提供 `_conf_schema.json` 用于生成配置表单。

必填配置：

- 和风天气 API Host
- 和风天气 API Key，或 JWT Project ID / Credential ID / Ed25519 Private Key
- 默认城市
- 天气日期切换时间，默认 `18:00`
- 日志输出级别，默认 `info`，可选 `debug` / `info` / `warn`
- 天气预警响应缓存时间，默认 10 分钟
- 家地址或所在区域
- 家坐标（可选，经度,纬度，例如 `116.41,39.92`）
- 公司地址或所在区域
- 公司坐标（可选，经度,纬度，例如 `116.41,39.92`）
- 上班通勤时间段，例如 `08:00-09:00`
- 下班通勤时间段，例如 `18:00-19:00`
- 可选通勤方式，例如 `地铁、公交、骑行、开车、打车`





## 天气预警

插件提供两个预警命令：

- `/预警`：手动查询当前生效预警，成功或失败都会回复；没有预警时回复“暂无生效天气预警”。
- `/预警检查`：适合定时插件调用，仅在发现新增或更新预警时输出；没有新预警时不输出任何消息。

预警 API 响应缓存时间可在插件配置页面调整，默认 10 分钟。预警去重状态固定保留 48 小时，并会在每次命令运行时清理过期响应缓存和过期去重状态，避免状态文件持续增长。

## 日志级别

插件配置页面提供“日志输出级别”下拉选项，默认 `info`，可选：

- `debug`：输出和风天气 API 请求信息和返回数据摘要，自动隐藏 API Key、JWT、私钥等敏感信息。
- `info`：默认等级，仅输出常规信息和警告。
- `warn`：仅输出警告。

## `/天气` 日期切换规则

默认天气日期切换时间为 `18:00`：

- 当前时间早于 `18:00` 时，`/天气` 输出今日详细天气。
- 当前时间达到或晚于 `18:00` 时，`/天气` 输出明日详细天气。

可在插件配置页面通过“天气日期切换时间”调整。

## 认证方式

插件支持两种和风天气认证方式：

1. JWT：推荐方式。填写 JWT Project ID、Credential ID 和 Ed25519 Private Key 后会优先使用 JWT。
2. API Key：兼容方式。JWT 三项全部留空时使用 API Key。

如果 JWT 只填写了一部分，插件会直接报错提示配置不完整，不会静默回退到 API Key。

## `/通勤` 智能判断表

以上班 `08:00-09:00`、下班 `18:00-19:00` 为例：

| 当前时间 | 输出内容 |
| --- | --- |
| 00:00-07:59 | 今日上班 + 今日下班 |
| 08:00-09:00 | 当前上班时段 + 今日下班 |
| 09:01-17:59 | 今日下班 |
| 18:00-19:00 | 当前下班时段 |
| 19:01-23:59 | 明日上班 + 明日下班 |

插件会先生成结构化天气/通勤数据，再调用 AstrBot 当前默认模型整理成更自然的聊天回复；模型不可用或超时时自动使用内置模板输出。

通勤天气会优先使用家坐标/公司坐标进行查询；坐标留空时回退到家地址/公司地址。坐标格式为和风天气支持的 `经度,纬度`。
